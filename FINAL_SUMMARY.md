## 🔧 技术实现思路

### **核心流程**
```
小爱音箱语音指令 → 小米云端API → 对话记录获取 → 小米Mimo模型解析 → WebSocket转发 → Chrome插件执行 → 浏览器操作
```

### **关键技术点**
1. **对话记录获取**：复用xiaomusic的轮询机制，定期从小米API获取对话记录
2. **AI指令解析**：使用小米Mimo模型的OpenAI兼容API，通过精心设计的提示词解析自然语言
3. **实时通信**：WebSocket提供双向实时通信，确保指令及时传递
4. **浏览器自动化**：Chrome插件使用多种API实现丰富的浏览器操作

## 🚀 快速开始

### **步骤1：部署系统**
```bash
cd xiaomi_mimo_browser_control
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### **步骤2：配置系统**
编辑 `backend/config.json`：
```json
{
  "xiaomi": {
    "username": "您的小米账号",
    "password": "您的小米密码"
  },
  "mimo_api": {
    "base_url": "https://api.xiaomimimo.com/v1",
    "api_key": "您的Mimo API Key",
    "model": "MiMo-V2-Flash"
  }
}
```

### **步骤3：启动服务**
```bash
./start.sh
```

### **步骤4：安装Chrome插件**
1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `browser_extension` 目录

### **步骤5：测试系统**

- "小爱同学，搜索最新新闻"
- "小爱同学，向下滚动"

## 🛠️ 故障排除

### **常见问题及解决方案**
1. **对话记录获取失败**
   - 检查小米账号密码
   - 确保小爱音箱已登录同一账号

2. **AI解析失败**
   - 检查Mimo API Key是否有效
   - 确保网络连接正常

3. **插件连接失败**
   - 确保后端服务正在运行
   - 检查WebSocket端口是否被占用

4. **浏览器操作失败**
   - 检查Chrome扩展权限设置
   - 确保目标页面已完全加载
